# check-unused-strings-ios

An AI agent skill that scans all Swift source files to find localization keys defined in `Strings.swift` (e.g. generated by [SwiftGen](https://github.com/SwiftGen/SwiftGen)) that are never referenced anywhere in the project. Reports unused keys and optionally removes them.

## How it works

1. The agent runs `scripts/checkUnusedStrings` from the project root.
2. The script locates `Strings.swift` automatically (searches the project, excludes `.build/` and `DerivedData/`).
3. It extracts every `public static let/func` member from `Strings.swift` as a defined key.
4. It greps all other Swift files for `Strings.<name>` references to build the used set.
5. The difference is reported as unused strings.
6. **Nothing is removed** until the user explicitly asks.
7. When removal is approved, the agent deletes the key from `Localizable.strings` and optionally runs `./buildStrings` to regenerate accessor files.

## Prerequisites

### `Strings.swift` (SwiftGen or equivalent)

The script expects a `Strings.swift` file in your project containing `public static let` or `public static func` members — the standard output of SwiftGen's `strings` template. If your project uses a different accessor pattern, the script will need adjustment.

### `buildStrings` script (optional)

If your project uses a codegen tool to generate `Strings.swift` from `Localizable.strings`, create a script named `buildStrings` in the project root. The agent runs it automatically after removing keys to keep generated files in sync.

Minimal example:

```bash
#!/bin/zsh
swiftgen  # or your codegen tool of choice
```

Make it executable:

```bash
chmod +x buildStrings
```

If no `buildStrings` script exists, the agent only edits `Localizable.strings` directly.

## Usage

Invoke the skill naturally:

```
check for unused strings
find dead localization keys
clean up unused strings before the release
are there any unused translation keys?
```

The agent runs the script, groups results by feature prefix, and waits for your instructions before removing anything.

## Multiple `Strings.swift` files

If your project has multiple modules each with their own `Strings.swift`, the script uses the first one it finds and warns you. For multi-module projects, run the skill once per module by `cd`-ing into each module root, or adapt the script to loop over all found files.

## AI agent integration

### Claude Code

Place this skill in `~/.claude/skills/check-unused-strings-ios/`. Claude Code will automatically load `SKILL.md` and trigger the skill whenever you ask to check or clean up unused strings.

### Codex / other agents

Point the agent to `scripts/checkUnusedStrings` in your project instructions (e.g. `AGENTS.md`) and instruct it to run the script from the project root and present the output before taking any action.
